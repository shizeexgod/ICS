# ICS — сайт-визитка сервиса автонапоминаний о записи

Одностраничный статичный сайт (HTML/CSS/JS без сборки и зависимостей) для сервиса **ICS**, который автоматически отправляет клиентам напоминания о записи в Telegram, WhatsApp и MAX.

## Структура

```
ICS/
├── index.html        # разметка страницы
├── css/style.css      # стили (тёмная/светлая тема)
├── js/script.js       # переключение темы, мобильное меню, анимации появления
├── assets/            # логотип
└── README.md
```

## Запуск

Никакой сборки не требуется — просто откройте `index.html` в браузере, либо поднимите локальный сервер:

```bash
npx serve .
```

## Контент

- **Блок 1 (hero):** название ICS, слоган про напоминания клиентам, лого.
- **Блок 2 (услуги):** автоматическая отправка уведомлений во все мессенджеры, работа через официальные аккаунты, интеграции с CRM, гибкие сценарии, уведомления об изменениях, сбор отзывов.
- **Блок «Мессенджеры»:** Telegram, WhatsApp, MAX.
- **Блок «Как это работает»:** 6 шагов подключения сервиса.
- **Блок 3 (контакты):** Telegram `@icsoffic1al`, телефон `+7 964 370-48-71`.

## Темы оформления

Сайт поддерживает 3 темы, переключаются кнопками-«таблетками» в хедере (сохраняются между визитами):

- **blue** — базовая синяя тема (по умолчанию);
- **light** — светлая, слегка сероватый фон, не бьёт по глазам;
- **dark** — глубокая нейтральная тёмная тема, близкая по духу к T.Technology.

Палитры описаны через CSS-переменные в `css/style.css` (`[data-theme="blue"]`, `[data-theme="light"]`, `[data-theme="dark"]`). Акцентные цвета мессенджеров (`--tg`, `--wa`, `--max`) общие для всех тем.

## Сохранение настроек (готово к бэкенду)

В `js/script.js` есть модуль `Settings` — асинхронная обёртка над хранением выбранной темы:

- по умолчанию использует `localStorage`;
- если на странице задать `window.ICS_API_BASE = "https://api.example.com"` (например, в `index.html` перед подключением `script.js`), модуль сам начнёт синхронизировать значение с бэкендом через `GET/PUT /settings/theme`, попутно оставляя `localStorage` как надёжный фолбэк при недоступности API.

Ожидаемый контракт эндпоинта:

```
GET  /settings/:key   -> { "value": "dark" }
PUT  /settings/:key   body: { "value": "dark" } -> 200 OK
```

Так к сайту можно подключить любой бэкенд (хранение предпочтений пользователя, аналитику и т.д.) без переписывания фронтенда.
